治理提醒

日常使用建议
  • 数据源仅向被授权用户开放，避免跨业务范围查询。
  • 账号、角色、部门统一维护，便于治理职责落地。
  • 健康检测与启停状态集中展示，降低无效查询入口。

# 电商业务库 — 业务语义说明

## 订单表 (orders)
- 存储所有交易订单，每条记录代表一次购买行为
- `total_amount`: 订单总金额（含折扣后），单位：元
- `status`: 订单状态（1=待付款, 2=已付款, 3=已发货, 4=已完成, 5=已取消）

## 常见查询模式
- "上月销售额" → SUM(total_amount) WHERE status IN (2,3,4)
- "退货率" → COUNT(status=5) / COUNT(*)
工作流： 元数据快照发布 → 自动生成 skills.md 草稿 → 人工审核编辑 → 发布 → 自动向量化入 Milvus → RAG 可召回
