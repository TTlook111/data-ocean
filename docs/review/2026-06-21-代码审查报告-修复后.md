## 🟡 需要优化的问题

### 问题1: ObjectMapper重复创建（中优先级）

**文件**: `PermissionCalculatorImpl.java:337`

**问题代码**:
```java
try {
    // 每次调用都创建新的ObjectMapper实例
    com.fasterxml.jackson.databind.ObjectMapper mapper = new com.fasterxml.jackson.databind.ObjectMapper();
    com.fasterxml.jackson.databind.JsonNode root = mapper.readTree(schedule);
```

**问题描述**:
- ObjectMapper 是线程安全的，应该作为类级别的常量
- 每次调用都创建新实例会造成不必要的对象创建开销

**修复建议**:
```java
// 在类定义中添加常量
private static final com.fasterxml.jackson.databind.ObjectMapper JSON_MAPPER = 
        new com.fasterxml.jackson.databind.ObjectMapper();

// 使用常量
try {
    com.fasterxml.jackson.databind.JsonNode root = JSON_MAPPER.readTree(schedule);
```

### 问题2: Controller返回Result.error()（低优先级）

**文件**: `NotificationController.java:56`

**问题代码**:
```java
@PatchMapping("/batch-read")
public Result<Void> markBatchAsRead(@RequestBody List<Long> ids) {
    if (ids == null || ids.isEmpty()) {
        return Result.success();
    }
    // 限制批量操作数量
    if (ids.size() > 100) {
        return Result.error("批量操作最多支持100条记录");  // 应该抛出异常
    }
```

**问题描述**:
- 返回 `Result.error()` 而不是抛出异常，可能导致前端无法正确处理错误
- 应该使用 `BusinessException` 抛出异常

**修复建议**:
```java
if (ids.size() > 100) {
    throw new BusinessException("批量操作最多支持100条记录");
}
```
