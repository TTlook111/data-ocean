## 已确认的开发任务

### P0 — 管理员反馈特权（反滥用机制补全）

**背景**：设计文档要求管理员/分析师的"踩"直接生效且权重为普通用户 3 倍，当前实现中所有用户走相同流程，未区分角色。

**改动范围**：

- `UserFeedbackServiceImpl.submitFeedback()` 增加角色判断逻辑
- 管理员/分析师的 DISLIKE 跳过审核队列，直接触发 `USER_DISLIKE_CONFIRMED` 事件，delta 使用 -45（普通用户 -15 的 3 倍）
- 普通用户流程不变（限流 → 审核队列 → 组阈值检测）

**涉及文件**：
- `backend/.../fieldtag/service/impl/UserFeedbackServiceImpl.java`
- `backend/.../fieldtag/service/impl/FeedbackReviewServiceImpl.java`（确认审核通过的逻辑正确）

### P1 — 通知系统前端接入

**背景**：后端通知系统已完整实现（`NotificationController` 提供 3 个 REST 端点），但前端零接入。

**改动范围**：

1. 新增 `frontend/src/api/notification.ts`，封装 3 个 API：
   - `GET /api/notifications` — 分页获取当前用户通知
   - `PATCH /api/notifications/{id}/read` — 标记已读
   - `GET /api/notifications/unread-count` — 未读数量

2. 在 `AppShell.vue` 顶部栏添加通知铃铛图标 + 未读角标 + 下拉通知列表

3. 可选：新增 `frontend/src/views/admin/system/NotificationList.vue` 通知列表完整页面

**涉及文件**：
- 新增 `frontend/src/api/notification.ts`
- 修改 `frontend/src/components/AppShell.vue`
- 可选新增 `frontend/src/views/admin/system/NotificationList.vue`

### P3 — 组阈值告警对接通知系统

**背景**：当 3 个不同用户在 7 天内对同一字段点"踩"时，系统会自动降级置信度（-5）并发布 `GroupThresholdAlertEvent`，但目前没有消费者监听这个事件。

**前置依赖**：P1（通知系统前端接入完成后再对接，管理员才能看到告警）

**改动范围**：

1. 在 `fieldtag` 模块新增 `GroupThresholdAlertEventListener`：
   - `@EventListener` 监听 `GroupThresholdAlertEvent`
   - 调用 `NotificationService.send()` 创建通知
   - 通知目标：具有 `field-tag:manage` 权限的管理员
   - 通知内容：字段名、表名、触发用户数、当前置信度分数

**涉及文件**：
- 新增 `backend/.../fieldtag/event/GroupThresholdAlertEventListener.java`
- 可能需要 `NotificationService` 增加按权限批量发送的方法

### P4 — PostgreSQL 数据源支持

**背景**：当前 MVP 仅支持 MySQL 数据源。后续需要扩展支持 PostgreSQL，让更多类型的业务数据库接入平台。

**改动范围**：

1. 数据源模块增加 PostgreSQL 数据库类型支持
2. 元数据采集器适配 PostgreSQL 的 `information_schema` 和 `pg_catalog`
3. SQL 沙箱适配 PostgreSQL 方言（sqlglot 已支持）
4. 连接池管理适配 PostgreSQL JDBC 驱动

---

### P5 — 数据血缘图数据库

**背景**：当前血缘数据存储在 MySQL 关系表中（`query_lineage_table`、`query_lineage_column`），查询多跳关系时效率低。后续考虑将血缘数据迁移到图数据库，提升血缘可视化和 RAG 准确性。

**改动范围**：

1. 引入图数据库（如 Neo4j 或 NebulaGraph）
2. 将表/列血缘关系写入图数据库
3. 血缘查询从 SQL JOIN 改为图遍历
4. 前端血缘可视化适配图数据库查询结果
