# 后台信息架构与导航规范

> 更新日期：2026-06-24

本文定义 DataOcean 后台管理端的信息架构、导航层级和新增页面归属规则。后续新增后台页面时，先按本文确认业务域和二级入口，再修改路由与导航，避免再次回到“功能页面平铺”的玩具化结构。

## 设计原则

DataOcean 后台不是功能页面集合，而是围绕“数据源从接入到可询问，再到持续治理”的工作系统。

主流程：

```text
数据源接入
-> 元数据采集
-> 治理校验
-> 语义知识构建
-> 权限开放
-> 可询问
-> 审计反馈
-> 持续优化
```

导航采用三层结构：

```text
一级：侧边栏业务域
二级：内容区工作区导航
三级：具体处理页面、详情页、弹窗或抽屉
```

侧边栏只回答“我在哪个业务域工作”，不承载所有功能入口。同步任务、快照、表浏览器、问题清单、字段标签、Prompt、审计日志等细页面，应放在对应业务域的二级工作区导航或页面内部动作中。

## 一级侧边栏

当前后台侧边栏由 `frontend/src/components/AppShell.vue` 的 `menuGroups` 定义，只保留业务域入口：

```text
总览
  工作台

数据源生命周期
  数据源中心
  治理中心
  语义中心

开放与运营
  权限与开放
  运营与安全
  系统设置
```

一级入口的权限可见性不是只看默认入口权限，而是看该业务域下是否存在任一当前用户可访问的二级页面。点击一级入口时，应进入该业务域下第一个可访问的二级页面。

## 二级工作区导航

二级导航由 `AppShell.vue` 的 `workspaceLinks` 定义，显示在内容区顶部，负责承接原来的具体功能入口。

### 数据源中心

归属对象：数据源、元数据采集、快照、目录与版本。

当前二级入口：

```text
数据源总览      /admin/datasources
同步任务        /admin/metadata/sync
快照列表        /admin/metadata/snapshots
表浏览器        /admin/metadata/tables
目录搜索        /admin/metadata/catalog
快照差异        /admin/metadata/diff
版本历史        /admin/metadata/version-history
同步调度        /admin/metadata/schedule
```

页面归属规则：

- 与数据源连接、采集、快照、表字段浏览、目录搜索、版本差异直接相关的页面放到数据源中心。
- 数据源详情页 `/admin/datasources/:id/lifecycle` 是数据源主页面，应通过数据源总览列表和数据源流程入口进入。
- 不要把新采集类页面直接加到侧边栏，应加到数据源中心二级导航或数据源详情页的阶段动作里。

### 治理中心

归属对象：质量校验、问题处理、治理状态、字段可信度和反馈闭环。

当前二级入口：

```text
治理总览        /admin/governance/quality
问题处理        /admin/governance/issues
治理状态        /admin/governance/status
快照生命周期    /admin/metadata/lifecycle
字段标签        /admin/field/tags
字段可信度      /admin/field/confidence
反馈闭环        /admin/field/feedback-review
```

页面归属规则：

- 与“数据是否可信、是否能进入 RAG/SQL 生成链路”相关的页面放到治理中心。
- 字段标签、字段可信度、用户反馈审核属于治理闭环，不再作为独立侧边栏一级模块。
- 治理中心页面应继承 `AdminContextBar` 的数据源/快照上下文。

### 语义中心

归属对象：skills.md、业务术语、Prompt 策略、知识审核与发布。

当前二级入口：

```text
知识文档        /admin/knowledge
文档编辑        /admin/knowledge/editor
知识审核        /admin/knowledge/review
术语体系        /admin/glossary/list
Prompt 策略     /admin/prompts
```

页面归属规则：

- 所有影响 AI 理解业务语义、SQL 生成策略或 RAG 内容的页面放到语义中心。
- Prompt 管理不是系统设置，而是语义链路的一部分。
- 知识版本页 `/admin/knowledge/versions/:id` 从知识文档或编辑页进入，不需要单独出现在二级导航。

### 权限与开放

归属对象：访问控制、策略、用户角色部门、未来的访问申请和审批队列。

当前二级入口：

```text
访问控制        /admin/permission/access
授权策略        /admin/permission/policies
用户            /admin/users
角色            /admin/roles
部门            /admin/departments
```

页面归属规则：

- 与“谁能问、能问什么、何时能问、如何审批”相关的页面放到权限与开放。
- 用户、角色、部门是权限支撑能力，不作为侧边栏一级入口。
- 后续开发“访问申请前台入口 + 审批队列页面”时，应归入此业务域；审批队列可作为二级入口，普通用户申请入口应优先从无权限提示或查询页发起。

### 运营与安全

归属对象：审计、慢查询、血缘影响、服务健康。

当前二级入口：

```text
审计日志        /admin/audit/logs
慢查询          /admin/audit/slow-queries
血缘查看        /admin/audit/lineage
血缘图谱        /admin/audit/lineage-graph
服务健康        /admin/system/health
```

页面归属规则：

- 查询行为、SQL 执行风险、血缘影响、运行健康属于运营与安全。
- 如果新增告警历史或安全风险页面，优先放入此域。

### 系统设置

归属对象：AI 配置、操作日志、基础系统配置。

当前二级入口：

```text
AI 配置         /admin/system/ai-config
操作日志        /admin/system/operation-logs
```

页面归属规则：

- 只有与平台全局配置、系统参数、管理员操作记录相关的页面放到系统设置。
- 告警规则、配额管理如果只是配置项，可放系统设置；如果主要服务运行监控和告警处理，应放运营与安全。

## 新增后台页面流程

新增后台页面时按以下顺序处理：

1. 判断页面服务哪个业务流程阶段：接入、采集、治理、语义、权限、运营、系统设置。
2. 放到对应 `frontend/src/views/admin/<domain>/` 目录；如果已有目录不匹配，应优先复用业务域目录而不是新建随意目录。
3. 在 `frontend/src/router/index.ts` 注册路由，并保持 `/admin/<domain>/...` 路径语义清晰。
4. 如页面需要二级入口，在 `AppShell.vue` 的 `workspaceLinks` 中添加，不直接加侧边栏一级入口。
5. 如页面只是详情、编辑、版本、弹窗承接页，优先从列表或详情页动作进入，不加二级导航。
6. 同步更新本文件、`AGENTS.md`、`CLAUDE.md` 和相关模块文档。

## 代码位置

当前导航实现位置：

```text
frontend/src/components/AppShell.vue
  - menuGroups: 一级侧边栏业务域
  - workspaceLinks: 二级工作区导航
  - routeMatches: 子页面归属匹配
  - navTarget: 一级入口点击时跳到第一个可访问二级页面
  - activeWorkspaceLink: 二级导航最长路径高亮
```

路由定义位置：

```text
frontend/src/router/index.ts
```

上下文条位置：

```text
frontend/src/components/AdminContextBar.vue
```

## 当前未完成但应遵循的方向

- 访问申请前台入口和审批队列页面尚需补齐，归属“权限与开放”。
- 各业务域首页还可以继续增强为真正的任务工作台，而不是只承接二级导航。
- 新增告警规则、告警历史、配额管理时，先判断它们是运营监控任务还是系统配置任务，再选择“运营与安全”或“系统设置”。
