## 二、发现的问题及修复

### 2.1 Milvus 连接问题

**问题**：LangChain Milvus 0.3.x 使用旧的 `Collection` API，与 `MilvusClient` 连接方式不兼容。

**错误信息**：
```
ConnectionNotExistException: (code=1, message=should create connection first.)
```

**修复方案**：
- 改用 `MilvusClient` 直接操作 Milvus
- 修改 `vector_store.py`，使用 `MilvusClient` 的 `insert`、`search`、`delete` 方法
- 修改 `milvus_client.py`，同时建立 `MilvusClient` 和旧的 `connections` 连接

**修改文件**：
- `python-service/dataocean/rag/vector_store.py`
- `python-service/dataocean/rag/milvus_client.py`

### 2.2 RAG 检索 table_name 为空

**问题**：`search_by_vector` 返回的 metadata 中 key 是 `table_name`，但 `retrieve_from_milvus` 使用 `related_table`。

**修复方案**：
- 修改 `retriever.py`，兼容两种 key：`table_name` 和 `related_table`

**修改文件**：
- `python-service/dataocean/rag/retriever.py`

### 2.3 SQL 校验分号检查过严

**问题**：SQL 末尾的分号（如 `SELECT * FROM orders LIMIT 1000;`）被误判为多语句注入。

**修复方案**：
- 修改 `validator.py`，只在分号后面有其他 SQL 语句时才拒绝（允许末尾分号）
- 修改 `parsers.py`，在提取 SQL 后去除末尾分号

**修改文件**：
- `python-service/dataocean/sandbox/validator.py`
- `python-service/dataocean/infra/parsers.py`

### 2.4 LLM 生成 SQL 包含分号

**问题**：LLM 生成的 SQL 可能包含末尾分号，被 SQL 校验拒绝。

**修复方案**：
- 修改 `sql_generation.j2` prompt，添加约束：禁止使用分号
- 修改 `parsers.py`，在提取 SQL 后去除末尾分号

**修改文件**：
- `python-service/dataocean/agent/prompts/sql_generation.j2`
- `python-service/dataocean/infra/parsers.py`

### 2.5 SQL Executor status 无效

**问题**：`sql_executor.py` 使用 `status='executing'`，但 `ProgressEvent` 只允许 `started`、`completed`、`failed`、`retrying`。

**错误信息**：
```
ValidationError: Input should be 'started', 'completed', 'failed' or 'retrying'
```

**修复方案**：
- 修改 `sql_executor.py`，将 `executing` 改为 `started`

**修改文件**：
- `python-service/dataocean/agent/nodes/sql_executor.py`

### 2.6 Decimal JSON 序列化问题

**问题**：MySQL 的 DECIMAL 类型被转换为 Python 的 Decimal，不能直接序列化为 JSON。

**错误信息**：
```
TypeError: Object of type Decimal is not JSON serializable
```

**修复方案**：
- 修改 `executor.py`，在转换为 dict 时将 Decimal 转换为 float
- 修改 `sse.py`，添加自定义 JSON 编码器处理 Decimal 类型

**修改文件**：
- `python-service/dataocean/sandbox/executor.py`
- `python-service/dataocean/infra/sse.py`

### 2.7 RAG 检索阈值过高

**问题**：默认相似度阈值 0.6 过高，导致检索结果被过滤。

**修复方案**：
- 修改 `config.py`，将 `similarity_threshold` 从 0.6 降低到 0.4

**修改文件**：
- `python-service/dataocean/core/config.py`
