# flows/ — 页面交互简报与端到端流程

本目录是 specs（架构边界）和实际实现之间的桥梁层。

## 用途

给 Codex / AI 编码工具提供足够具体的实施指令，包括：
- 每个页面的所有状态（空、加载、正常、错误、边界）
- 关键交互的具体响应行为
- 与上下游页面的衔接关系
- 视觉和体验的具体要求
- 端到端用户旅程（跨模块串联）

## 文件结构

```
flows/
├── README.md
├── e2e-journeys.md          — 跨模块端到端用户旅程
├── 001-user-module.md       — 用户模块所有页面交互简报
└── 002-datasource.md        — 数据源模块所有页面交互简报
```

## 如何使用

1. 开发新页面前，先读对应的交互简报
2. 联调时对照简报检查所有状态是否覆盖
3. 发现简报遗漏的场景，及时补充
