# 001 用户模块 — 页面交互简报

---

## 页面 1：登录页 `/login`

### 布局
左右分栏（6:4）。左侧品牌区深色背景 + 产品介绍，右侧白色登录表单。移动端隐藏左侧，表单居中。

### 表单字段
| 字段 | 类型 | 校验规则 | 校验时机 |
|------|------|----------|----------|
| 用户名 | text | 必填，2-50字符 | blur + submit |
| 密码 | password | 必填，8-32字符 | submit only |
| 记住我 | checkbox | — | — |

### 状态矩阵

| 状态 | 表现 |
|------|------|
| 初始 | 表单空白，登录按钮可点击，光标聚焦用户名输入框 |
| 输入中 | 实时清除上一次的错误提示 |
| 提交中 | 按钮显示 loading + "登录中..."，表单字段 disabled |
| 成功 | 按钮变绿 0.3s → 跳转目标页 |
| 失败-凭证错误 | 表单上方红色 alert："用户名或密码错误（剩余N次尝试）" |
| 失败-账号锁定 | alert："账号已锁定，请联系管理员" |
| 失败-网络错误 | alert："网络连接失败，请检查网络后重试" |
| 失败-服务端错误 | alert："服务暂时不可用，请稍后重试" |

### 交互细节
- Enter 键提交表单
- "记住我"勾选后，下次打开页面自动填充用户名（localStorage）
- 密码输入框右侧有显示/隐藏切换图标
- 登录成功后跳转逻辑：有 redirect 参数则跳转，管理员跳 `/admin`，普通用户跳 `/query`
- 连续失败 5 次后账号锁定，第 3 次起显示剩余次数警告

### 上下游
- 来源：直接访问 / 退出登录 / token 过期被踢出
- 去向：管理首页 / 问答端 / 强制改密页

## 页面 2：强制改密页 `/change-password?forced=1`

### 布局
居中卡片式布局，宽度 400px。顶部提示文案区 + 表单 + 按钮。

### 表单字段
| 字段 | 类型 | 校验规则 | 校验时机 |
|------|------|----------|----------|
| 当前密码 | password | 必填 | submit |
| 新密码 | password | 必填，8-32字符，含字母+数字 | blur + submit |
| 确认密码 | password | 必填，与新密码一致 | blur + submit |

### 状态矩阵

| 状态 | 表现 |
|------|------|
| 强制模式 | 顶部黄色提示条："首次登录，请修改初始密码后继续使用" |
| 自愿模式 | 无提示条，标题为"修改密码" |
| 密码强度反馈 | 新密码输入时下方显示强度条（弱/中/强） |
| 提交中 | 按钮 loading |
| 成功 | Message 提示"密码修改成功，请重新登录" → 1.5s 后跳转登录页 |
| 失败-旧密码错误 | "当前密码不正确" 错误提示在对应字段下方 |
| 失败-新密码不合规 | 字段下方红色提示具体原因 |

### 交互细节
- 强制模式下不显示侧边栏和顶栏（独立页面布局）
- 强制模式下浏览器后退按钮无效（router guard 拦截）
- 新密码不能与当前密码相同
- 确认密码实时校验一致性（输入时即比对）

### 上下游
- 来源：登录成功但 passwordChanged=false 时自动跳转
- 去向：修改成功 → 登录页（清除 token）

## 页面 3：管理首页 `/admin`

### 布局
顶部欢迎区 + 模块状态卡片网格 + 快捷入口 + 开发进度面板。

### 状态矩阵

| 状态 | 表现 |
|------|------|
| 正常 | 显示各模块状态卡片，已完成的标绿，进行中标蓝，未开始标灰 |
| 加载中 | 骨架屏（skeleton）占位 |
| 无权限模块 | 卡片仍显示但置灰，hover 提示"无权限" |

### 交互细节
- 欢迎语包含用户姓名和当前时间段（早上好/下午好/晚上好）
- 模块卡片点击跳转对应管理页
- 快捷入口根据用户权限动态显示

### 上下游
- 来源：管理员登录成功 / 侧边栏点击"工作台"
- 去向：各管理页面

## 页面 4：用户管理 `/admin/users`

### 布局
顶部筛选工具栏 + 数据表格 + 底部分页器。右上角"新建用户"按钮。

### 筛选工具栏
| 筛选项 | 类型 | 行为 |
|--------|------|------|
| 用户名 | text input | 模糊搜索，防抖 300ms |
| 姓名 | text input | 模糊搜索，防抖 300ms |
| 部门 | tree-select | 选择后立即过滤，含子部门 |
| 状态 | select | NORMAL/DISABLED/LOCKED，支持"全部" |
| 重置按钮 | button | 清空所有筛选条件，重新加载 |

### 表格列
| 列 | 宽度 | 特殊渲染 |
|----|------|----------|
| 用户名 | 120px | 固定列 |
| 姓名 | 100px | — |
| 部门 | 120px | 显示部门路径（如"技术部/后端组"） |
| 角色 | 150px | 多个角色用 Tag 展示，超过2个折叠+tooltip |
| 状态 | 80px | 彩色 Tag：正常=绿，禁用=灰，锁定=红 |
| 最后登录 | 160px | 相对时间（"3小时前"），hover 显示完整时间 |
| 操作 | 200px | 固定列，按钮组 |

### 操作按钮逻辑
| 按钮 | 条件 | 确认 | 成功反馈 |
|------|------|------|----------|
| 编辑 | 始终可见 | 无 | 打开编辑弹窗 |
| 启用 | 状态=DISABLED | 无 | 行内状态即时更新 |
| 禁用 | 状态=NORMAL | 确认弹窗 | 行内状态即时更新 |
| 解锁 | 状态=LOCKED | 确认弹窗 | 行内状态变为 NORMAL |
| 重置密码 | 始终可见 | 确认弹窗+显示新密码 | 弹窗显示临时密码，可复制 |
| 删除 | 非当前用户 | 确认弹窗（红色警告） | 行消失，总数-1 |

### 新建/编辑弹窗
| 字段 | 新建 | 编辑 | 校验 |
|------|------|------|------|
| 用户名 | 可编辑 | 只读 | 必填，2-50字符，字母数字下划线 |
| 密码 | 必填 | 隐藏 | 8-32字符 |
| 姓名 | 可编辑 | 可编辑 | 必填，2-20字符 |
| 邮箱 | 可编辑 | 可编辑 | 选填，邮箱格式 |
| 手机 | 可编辑 | 可编辑 | 选填，11位手机号 |
| 部门 | tree-select | tree-select | 必填 |
| 角色 | multi-select | multi-select | 至少选一个 |

### 状态矩阵

| 状态 | 表现 |
|------|------|
| 加载中 | 表格显示 skeleton 行（5行占位） |
| 空数据 | 表格区域显示空状态插图 + "暂无用户数据" |
| 筛选无结果 | "未找到匹配的用户，试试调整筛选条件" + 重置按钮 |
| 网络错误 | 表格区域显示错误状态 + 重试按钮 |
| 弹窗提交中 | 确认按钮 loading，表单 disabled |
| 删除确认 | 红色警告弹窗："确定删除用户 {username}？此操作不可恢复" |

### 交互细节
- 新建成功后表格自动刷新并定位到第一页
- 编辑成功后当前行数据原地更新（不跳页）
- 删除后如当前页无数据则自动回退上一页
- 分页：默认每页 20 条，可选 10/20/50
- 表格行 hover 高亮
- 操作按钮根据状态动态显示（不是全部显示再 disabled）

### 上下游
- 来源：侧边栏"用户管理" / 管理首页快捷入口
- 去向：无直接跳转（所有操作在当前页完成）

## 页面 6：部门树 `/admin/departments`

### 布局
左侧树形结构（占满高度），节点右侧操作按钮。顶部"新建根部门"按钮。

### 树节点渲染
```
📁 部门名称 [排序号]          [添加子部门] [删除]
  📁 子部门名称 [排序号]      [添加子部门] [删除]
```

### 状态矩阵
| 状态 | 表现 |
|------|------|
| 空 | "暂无部门，点击上方按钮创建" |
| 正常 | 树默认全部展开 |
| 加载中 | 树区域 skeleton |
| 删除确认 | "确定删除部门 {name}？" + 如有子部门额外警告 |

### 新建/编辑弹窗
| 字段 | 校验 |
|------|------|
| 上级部门 | tree-select，新建根部门时为空 |
| 部门名称 | 必填，2-50字符 |
| 部门编码 | 必填，字母数字，唯一 |
| 排序号 | 数字，默认0 |

### 交互细节
- 删除部门时如果有子部门，提示"该部门下有 N 个子部门，将一并删除"
- 删除部门时如果有关联用户，提示"该部门下有 N 个用户，请先转移用户"并阻止删除
- 添加子部门时自动填充上级部门为当前节点
- 新建成功后树自动展开到新节点位置

### 上下游
- 来源：侧边栏"部门管理"
- 去向：无（但部门数据被用户管理页的部门选择器引用）

## 页面 7：个人资料 `/profile`

### 布局
居中卡片，宽度 600px。头部用户头像区 + 表单 + 底部操作区。

### 表单字段
| 字段 | 可编辑 | 校验 |
|------|--------|------|
| 用户名 | 否（灰色展示） | — |
| 姓名 | 是 | 必填，2-20字符 |
| 邮箱 | 是 | 选填，邮箱格式 |
| 手机 | 是 | 选填，11位 |

### 状态矩阵
| 状态 | 表现 |
|------|------|
| 初始 | 表单填充当前用户信息 |
| 修改中 | 保存按钮高亮，出现"取消"按钮 |
| 保存中 | 按钮 loading |
| 保存成功 | Message 提示"保存成功"，顶栏用户名同步更新 |
| 无修改 | 保存按钮 disabled |

### 交互细节
- 表单有变更检测，未修改时保存按钮不可点
- 离开页面时如有未保存修改，弹出确认弹窗
- "修改密码"链接在表单下方，跳转 `/change-password`（非强制模式）
- 保存成功后更新 Pinia store 中的用户信息（顶栏同步）

### 上下游
- 来源：顶栏用户名点击 / 侧边栏"个人资料"
- 去向：修改密码页
