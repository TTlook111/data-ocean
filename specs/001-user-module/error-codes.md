# 统一错误码规范

## 格式

错误码格式：`{MODULE}{SEVERITY}{SEQ}`
- MODULE: 3位模块标识（USR=用户, DSC=数据源, MTA=元数据, GOV=治理, KNW=知识库, RAG=召回, AGT=Agent, SQL=沙箱, TAG=可信度, AUD=审计, QRY=查询, SYS=系统）
- SEVERITY: 1位（4=客户端错误, 5=服务端错误）
- SEQ: 3位序号

## 用户模块错误码

| Code | Message | 场景 |
|------|---------|------|
| USR4001 | 用户名或密码错误 | 登录失败 |
| USR4002 | 账号已被禁用 | 禁用用户尝试登录 |
| USR4003 | 账号已被锁定 | 锁定用户尝试登录 |
| USR4004 | 用户名已存在 | 创建重复用户名 |
| USR4005 | 旧密码错误 | 修改密码时旧密码不对 |
| USR4006 | 密码不符合复杂度要求 | 密码太简单 |
| USR4007 | 不允许删除超级管理员 | 尝试删除 admin |
| SYS4001 | 参数校验失败 | JSR-303 校验不通过 |
| SYS4002 | 未登录或 Token 已过期 | 401 |
| SYS4003 | 权限不足 | 403 |
| SYS5001 | 系统内部错误 | 未知异常 |
